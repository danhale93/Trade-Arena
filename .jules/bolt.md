## 2025-06-15 - Redundant UI and Background Persistence
**Learning:** The Trade Arena codebase exhibited a pattern where expensive UI updates (O(N) calculations, SVG rendering, and DOM manipulation) were performed for a hidden "Quant Report" panel on every trade closure. Additionally, the PWA was subject to process suspension on mobile devices when locked, silencing audio telemetry.
**Action:** Always implement early return checks in UI update functions to verify visibility (e.g., via CSS classes like `open`) before executing heavy computations. Use the **Media Session API** and **Wake Lock API** to maintain process activity and audio continuity for "monitor-and-listen" applications in background states.
